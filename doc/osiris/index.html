ALPS Project: osiris library

The osiris library contains code for (de)serialization and message passing.

We aim for the serialization part to be soon superseded by a Boost library.

Download, Configuring, Building and Installing

are discussed here.

The following libraries will be built and installed . The libraries using message passing will only be built if the underlying message passing libraries are available and have been specified in the configuration step.

  • The serialization library : libosiris.a
  • The dummy communication library for non-parallel programs : libcomm-sgl.a
  • The communication library using MPI : libcomm-mpi.a
  • The communication library using PVM : libcomm-pvm.a

Documentation

  • OS-specific libraries
    • os.h low level operating system call wrappers
  • Serialization
    • dump.h contains the base classes for the osiris serialization mechanism.
    • xdrdump.h contains classes for serialization (e.g. into binary files) using the XDR format
  • Message passing
    • process.h contains classes describing processes and host machines.
    • comm.h contains wrappers to low level message passing routines.
    • mpdump.h contains classes for serialization into messages.
  • Support of other libraries
    • stringvalue.h contains serialization for palm::StringValue.
    • parameters.h contains serialization for palm::Parameters.
    • parameterlist.h contains serialization for palm::Parameterlist.
    • std/pair.h contains serialization for std::pair.
    • std/vector.h contains serialization for std::vector.
    • std/valarray.h contains serialization for std::valarray.
    • std/map.h contains serialization for std::map.
    • std/string.h contains serialization for std::string.
