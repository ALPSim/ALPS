ALPS Project: lattice library

For the simulation of lattice models the graph concepts of the Boost Graph Library (BGL) provide a good foundation. However, as discussed with the corresponding lattice schema at http://xml.comp-phys.org/. We want to be able to automatically create a "lattice" graph by translating a "unit cell".

This library defines concepts, traits, generic functions and some concrete classes to implement this idea. Lattices, Finite lattices, unit cells, and graphs as presented for the lattice schema can be created from an XML description, and a graph generated from them. The generic concepts presented below make the library extensible.

The lattice library contains code for the automatic generation of a graph from a lattice description.

The usual usage scenario will be to create a library of lattices and graphs by reading an XML file. The library can then be asked to create a graph, if one passes the graph name and optionally parameters. We refer to the example for a self-explaining usage example.

Documentation

Concepts

  • Helper concepts
    • Dimensional
    • FixedDimensional
    • Point
    • Vector
    • Coordinate
    • Refinements and combinations
  • Lattice concepts
    • UnitCell
    • Cell
    • Lattice
    • CoordinateLattice
    • FiniteLattice
  • Concepts for lattices with graphs
    • LatticeGraph
    • UnitCellGraph and GraphUnitCell
    • ReflectablePropertyGraph

Headers

  • General header
    • lattice.h includes all headers of the lattice library.
  • Traits classes
    • lattice/dimensional_traits.h defines traits and functions for the Dimensional concept.
    • lattice/point_traits.h defines traits for the Point concept.
    • lattice/coordinate_traits.h defines traits and functions for the Coordinate concept.
    • lattice/cell_traits.h defines traits for the Cell concept.
    • lattice/graph_traits.h defines traits and functions for the LatticeGraph and UnitCellGraph concepts.
  • Generic functions and template classes
    • lattice/graph_traits.h map the BGL notation to that of lattice models.
    • lattice/graphproperties.h defines property types used in the lattice library.
    • lattice/graph.h defines functions for XML I/O, copying and some property calculation functions of graphs.
    • lattice/parity.h functions for bipartite graphs.
    • lattice/lattice.h defines some basic functions for lattices.
    • lattice/latticegraph.h defines a function to create a graph from a lattice, and a template class that decorates a lattice with a graph.
  • Classes
    • lattice/unitcell.h defines two unit cell classes: EmptyUnitCell and GraphUnitCell
    • lattice/simplecell.h defines a template class simple_cell making a cell from a unit cell.
    • lattice/simplelattice.h defines a template class simple_lattice making a lattice from a cell and unit cell.
    • lattice/coordinatelattice.h defines a template class coordinate_lattice which adds concrete basis vectors to a lattice.
    • lattice/hypercubic.h defines a template class hypercubic_lattice which makes a hypercubic finite lattice with open or periodic boundary conditions from an infinite lattice.
    • lattice/latticedescriptor.h defines lattice classes which can be constructed from <LATTICE> and <FINITELATTICE> elements of XML input.
    • lattice/latticegraphdescriptor.h defines a class which can be constructed from <LATTICEGRAPH> elements of XML input and can be used to create graphs.
    • lattice/latticelibrary.h defines a class to hold a colection of graphs and lattices, read from XML, and a factory to create graphs from this collection.

Examples

To build the examples go to the test/lattice subdirectory of ALPS directory and type 
make examples
The source code and of the examples is well documented and self-explanatory.

To run the examples type 

./example1 < example1.input
./example2 < example2.input
./example3 < example3.input
./example4
./parity < parity.input
